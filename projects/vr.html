VR Rhythm-Action Game

The Project
A VR rhythm-action game for Meta Quest 3 where players use controller-tracked sabers to categorize incoming passwords as strong or weak — with an integrated quiz layer that tests cybersecurity knowledge between gameplay segments. Developed professionally at Zinad.
SDK Migration
Migrated the project from Oculus Integration SDK 0.46.0 to Meta XR All-in-One SDK 72.0.0. Rebuilt the player prefab using Building Blocks (controller-only, no hand tracking), rewrote broken hand-tracking API calls across multiple systems, and resolved Android build failures caused by a TMP3D shader incompatibility with GLES3 multiview stereo rendering.
Gameplay & Physics
Implemented a dual-saber hit detection system using Unity's trigger collider pipeline. Passwords spawn from an object pool and travel toward the player; each password carries a CollisionHandler that classifies hits as correct or incorrect based on saber/password tag combinations.
  • Solved a same-frame double-hit problem where Unity queues multiple OnTriggerEnter callbacks in a single physics step before any callback can execute
  • Fixed by introducing a hit guard bool on the password side coordinated with a WasHit property on the saber
  • Script Execution Order enforced to guarantee collision processing sequence
Scoring & Combo System
Built a ScoreKeeper MonoBehaviour tracking score, combo multiplier, and hits-in-a-row, exposing a WhenChanged C# event for reactive UI. Integrated a ScriptableObject-based score variable (IntSO) that fires change events independently of GameObject active state — solving a persistent UI update bug where ScoreCardUI was subscribing to the wrong ScoreKeeper instance after quiz answers. Scoring applies a velocity multiplier using a per-frame saber velocity tracker, rewarding faster swings with bonus points.
Audio Sync
Implemented a DSP clock-based password spawner that syncs object pool releases to the BPM of the selected track, keeping gameplay tightly coupled to the music regardless of frame rate variation.
Quiz System
Built a QuizManager loading questions from a JSON file in StreamingAssets using UnityWebRequest for Android compatibility. Implemented a crossfade transition using DOTween — screen fades to black, state machine switches to quiz state, panel fades in, answer flashes green or red, then reverses on exit. Supports Arabic and English with live locale switching mid-session.
State Machine & Architecture
Used a ScriptableObject-based state machine to switch between gameplay and quiz states, with BoolSO variables driving spawner pause/resume and IntSO variables driving combo and score display across decoupled UI components. All cross-system communication runs through the SO data layer — UI, spawning, scoring, and audio react to SO variable changes rather than direct references, keeping systems loosely coupled across scene boundaries.
