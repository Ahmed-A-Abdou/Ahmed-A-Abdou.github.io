Firebase Inventory System

The Goal
The goal was to build a real-time inventory system in Unity that syncs with Firebase — and to do it in a way that was genuinely reusable. Not a one-off script tied to a specific game, but a modular backend layer that any Unity project could plug into with minimal changes.
What I Built
  • Real-time inventory system with live syncing to Firebase Realtime Database
  • Data storage and retrieval with clean read/write abstraction layer
  • Modular backend-connected architecture separating data logic from gameplay logic
  • Reusable system components designed to drop into other Unity projects
What I Learned
Working with Firebase taught me how to think about data flow between a game client and an external backend — when to read, when to write, and how to handle async operations without breaking gameplay. I also got a much clearer picture of what "modular" actually means in practice: it's not just about separating files, it's about designing APIs between your systems so they don't need to know how the other side works.
