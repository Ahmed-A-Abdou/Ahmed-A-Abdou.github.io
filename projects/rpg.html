RPG Battle System Prototype

The Goal
Build a turn-based RPG combat system that was genuinely extensible — where adding a new ability, enemy type, or status effect wouldn't require touching existing code. The prototype was designed as a foundation that a full game could be built on top of without needing to rewrite the combat layer.
What I Built
  • Turn-based combat loop with clean state management between player and enemy turns
  • Modular combat system architecture separating actions, entities, and UI
  • State machine controlling battle flow: Idle, PlayerTurn, EnemyTurn, Victory, Defeat
  • Action system designed to be extended with new abilities without modifying core logic
  • Data-driven approach so enemy stats and abilities live outside code
What I Learned
This prototype taught me how to apply the Open/Closed principle in a real game context — designing systems that are open for extension but closed for modification. State machines clicked for me here in a way they hadn't before, because battle flow is a natural fit for them. I also practiced separating data from behaviour, which made testing individual combat scenarios much faster and cleaner.
