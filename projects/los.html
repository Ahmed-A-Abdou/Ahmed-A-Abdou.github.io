Line-of-Sight Monster Mechanic

The Goal
Build an AI detection system based on actual visibility — not proximity or simple triggers — so that a monster reacts to the player only when it can genuinely see them. The challenge was making this feel accurate and responsive while being efficient enough to run well at runtime.
What I Built
  • Visibility detection using raycasting to check for unobstructed line of sight to the player
  • Frustum-based field of view check so the monster only reacts within its visible angle
  • Layered detection logic combining both checks for accurate results
  • Optimized update frequency to avoid expensive checks every frame
  • Camera-based mechanic integration tying visibility to player observation
What I Learned
This project gave me a solid practical understanding of raycasting beyond basic collision — specifically using it for spatial reasoning in AI. I learned how frustum math works in 3D space and why combining angle checks with raycast checks catches edge cases that either method alone would miss. It also reinforced the value of optimizing detection logic: running expensive checks less frequently while keeping the game feeling responsive is a real balancing act.
