ColorSwap

The Game
A 2D mobile puzzle game where players resolve a graph-coloring problem disguised as a swap mechanic. Nodes connected by edges must never share a color with a directly connected neighbor — players drag colored tokens onto adjacent nodes to swap colors until the graph is properly colored.

The win condition is a proper graph coloring: the system walks every connection in the level and fails if any two connected nodes share a color. Swaps are only legal between directly connected nodes, so puzzle design boils down to picking node layouts and color assignments that create solvable coloring conflicts.
Data-Driven Level Authoring
Levels are ScriptableObjects holding a list of nodes (viewport-normalized position + starting color) and index-pair edge connections. New levels can be authored entirely as assets without touching gameplay code.

Node positions convert from normalized viewport space to world space via Camera.main.ViewportToWorldPoint, keeping layouts resolution-independent across all Android screen sizes automatically.
Decoupled Architecture
Systems communicate through static C# events wired in OnEnable/OnDisable. No system holds a direct reference to another — input detection, swap validation, and UI stay fully decoupled through a lightweight pub/sub layer. The result is a codebase where each system can be modified or replaced without touching anything else.
Interaction & Feedback
Drag detection uses Physics2D.OverlapPoint against oversized colliders for forgiving touch input — small targets feel large in hand. Invalid swaps and drag cancellations both resolve through a DOTween Ease.OutBack return animation, giving the player clear physical feedback without hard snapping. Tween configuration is centralized in a static TweenHelper wrapper so animation behaviour is consistent across the entire game and can be tuned from one place.
