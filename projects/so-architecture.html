ScriptableObject Architecture System

The System
A lightweight event and variable system that eliminates direct script dependencies in Unity projects. Systems communicate through ScriptableObject assets — any script can raise an event or read a shared variable without knowing who's listening or who set the value. No singletons, no service locators, no tight coupling.
Variable Library
A full typed variable library covering the most common Unity data types:
  • IntSO, FloatSO, BoolSO, StringSO
  • Vector3SO, GameObjectSO
Each variable asset holds a runtime value and a default value. On scene load or game reset, calling ResetToDefault() restores the asset to its original state — no manual cleanup code required. Variables fire a C# event on every value change, so any listener updates automatically without polling.
Event System
Asset-driven events wirable entirely through the Inspector without writing any connection code. A GameEventSO asset acts as the channel — raisers hold a reference and call Raise(), listeners subscribe via a GameEventListener component that invokes a UnityEvent in response. Adding or removing a listener is a drag-and-drop operation in the Inspector, keeping system wiring visible and editable by designers without touching code.
In Production
Currently used as the communication backbone in Cam3. Every inter-system message — enemy state changes, battery drain events, camera switches, player input signals — routes through this layer. No system in Cam3 holds a direct reference to another; all cross-system communication goes through SO variables and events, making each system independently testable and replaceable.
